## 1. Signal Chain Overview

```
L_IN ──┐
R_IN ──┤  [A: Input Buffer]  clamp ±11 V; R normalises to L when unpatched
       │  [1: Pre-Gain]      GAIN_MAIN → 1× pass or 5× (clip ±10.5 V)
       │  [VCA]              VCA_AMT + VCA_OFS; CV from VCA_IN (mod bus when unpatched)
       │  [LP1]              2-pole SVF LP; FREQ + stereo TILT + RES
       │
       └──────────────────────────────────────────────────── ← dry signal into BP_MIX
                                                              ↕ BP_MIX (0=dry, 1=wet)
ALT_BP_L/R → GAIN_BP3 → ────────────┐
                                     ↓
                          [BP]
                            global: BP_OFFSET (V/oct), BP_TILT (stereo CV)
                            per group (×3): FREQ + FOCUS (Q) + DIST (drive) + DIST mode (Soft/Hard/Fold)
                            formant ref: BP1=200 Hz, BP2=1500 Hz, BP3=6000 Hz
                            BP3_L/R_OUT ← tap here (after BP3 distortion)
                            summed (1× each, no weighting) → BP_MIX wet path

       ┌── BP_MIX output ─────────────────────────────────────────────────────────────┐
       │  [HP]  2-pole SVF HP; HP_FREQ (slider) + HP_RES                             │
       │  [LP2] 2-pole SVF LP; LP2_FREQ (slider) + LP2_RES                           │
       │  [B: Output Buffer]  clamp ±11 V → MAIN_L_OUT / MAIN_R_OUT                 │
       └──────────────────────────────────────────────────────────────────────────────┘

LFO1 / LFO2 ── triangle ±5 V; both feed MOD_SRC switch (LFO1 / LFO2 / External)
Mod Bus      ── MOD_SRC → SCALE (×0.2–5) → OFFSET (±5 V) → clamp ±10 V
               mod bus normalises → VCA_IN when VCA_IN unpatched
```

### ALT input routing
When `ALT_BP_L` is patched: the ALT path (post GAIN_BP3 pre-gain) replaces the LP1 output as the
BP section input. `ALT_BP_R` normalises to `ALT_BP_L` when unpatched but `ALT_BP_L` is present.
VCA and LP1 are still live (their output is the dry side of BP_MIX) but bypassed by the ALT path
feeding the BP wet computation.

## 6. Per-Block DSP Behaviour

### Block A — Input Buffer
```
inL = clamp(L_IN, −11, +11)
inR = R_IN connected ? clamp(R_IN, −11, +11) : inL
```

### Block 1 — Pre-Gain
```
gainMain = GAIN_PARAM >= 0.5
pgL = gainMain ? clamp(inL × 5, −10.5, +10.5) : inL
pgR = gainMain ? clamp(inR × 5, −10.5, +10.5) : inR
```

ALT path (runs in parallel; goes through the VCA, bypasses LP1; feeds **BP3 only**):
```
gainBP3 = ALT_GAIN_PARAM >= 0.5
altL = ALT_BP_L connected ? (gainBP3 ? clamp(ALT_BP_L × 5, −10.5, +10.5) : ALT_BP_L) : 0
altR = ALT_BP_R connected ? (gainBP3 ? clamp(ALT_BP_R × 5, −10.5, +10.5) : ALT_BP_R)
                          : (ALT_BP_L connected ? altL : 0)
```
Note: the ALT signal is then passed through the VCA (same VCA_AMT/CV as the main path) and
becomes the **BP3 band input only** — BP1/BP2 always use the main post-VCA/LP1 band. If
ALT_BP_L is unpatched, BP3's left input stays on the main band (`altL` = 0 is not injected);
if only `ALT_BP_L` is patched, `altR = altL`. (See `Pogo.cpp:344–352, 439–441`.)

### LFOs
```
lfo1V = LFO.process(LFO1_RATE_PARAM, dt) × 5   // ±5 V triangle
lfo2V = LFO.process(LFO2_RATE_PARAM, dt) × 5
LFO1_LIGHT = (lfo1V / 5 + 1) × 0.5             // 0–1 brightness
LFO2_LIGHT = (lfo2V / 5 + 1) × 0.5
```
Rate taper (unchanged from old code): `f = 0.05 × 400^param` Hz

One-pole LP at 10× the LFO rate is applied to model hardware integrator slew and Schmitt peak rounding.

### Mod Bus
```
// MOD_SRC_PARAM is a 3-way switch: 0 = LFO 1, 1 = LFO 2, 2 = External (MOD_IN)
srcV = (MOD_SRC == 0) ? lfo1V
     : (MOD_SRC == 1) ? lfo2V
     : (MOD_INPUT connected ? MOD_INPUT : 0)      // External: MOD_IN jack only, 0 V if unpatched
busV = ModBusProcessor::process(srcV, MOD_SCALE_PARAM, MOD_OFFSET_PARAM)
     = clamp(srcV × (0.2 × 25^scale) + offset × 5, −10, +10)

MOD_CLIP_LIGHT = |busV| >= 9.9 ? 1 : 0
MOD_POS_LIGHT  = busV > 0 ? busV / 10 : 0
MOD_NEG_LIGHT  = busV < 0 ? −busV / 10 : 0
```

### Block VCA
```
vcaCVraw = VCA_INPUT connected ? VCA_INPUT : busV
vcaCV    = clamp(vcaCVraw + VCA_OFS_PARAM × 5, 0, 10)   // OFS shifts floor
vcaOut   = VcaBlock::process(pgL/pgR, VCA_AMT_PARAM, vcaCV)
```

VCA gain law (dB-law, from `VcaBlock.hpp`):
```
normCV   = clamp(vcaCV / 5, 0, 1)                           // 0–1 from 0–5 V
control  = (amtParam >= 0) ? 1 − amtParam×(1−normCV) : 1 + amtParam×normCV
G        = 10^(2×(control−1))   // dB-law: 0 dB at control=1, −40 dB at control=0
```
- `AMT = 0` → control = 1 → G = 1 (unity always)
- `AMT > 0` → accent: control = 1 at CV=5, 1−AMT at CV=0 (mute when AMT=1 and CV=0)
- `AMT < 0` → duck:   control = 1 at CV=0, 1+AMT at CV=5 (mute when AMT=−1 and CV=5)

### Block LP1
```
// Attenuverter helper (same as old modDest()):
// cv_dest(param_att, cv_in) = busV × att + (cv_in connected ? cv_in × att : 0)
// Simplified: combined_cv = att × (cv_in connected ? cv_in : busV)

lp1FreqCv = lp1FreqBase + att(LP1_FREQ_ATT, LP1_FREQ_INPUT)   // V/oct
lp1TiltCv = LP1_TILT_PARAM × 5 + att(LP1_TILT_ATT, LP1_TILT_INPUT)  // ±5 V
lp1ResCv  = clamp(LP1_RES_PARAM + att(LP1_RES_ATT, LP1_RES_INPUT) / 10, 0, 1)   // CV ÷10 onto 0–1

lp1L = LPFilter::process(vcaOutL, lp1FreqCv + lp1TiltCv, lp1ResCv, sampleRate)
lp1R = LPFilter::process(vcaOutR, lp1FreqCv − lp1TiltCv, lp1ResCv, sampleRate)
```

LP1 filter parameters (unchanged from `LPFilter.hpp`):
- f_ref = 632 Hz; ω₀ = 2π × 632 × 2^(freqCv)
- Q taper: `Q = 0.5 × 4000^resParam`

### Block BP (Simper trapezoidal SVF)

**Pre-computation (once per base-rate sample):**
```
// Global BP CV
bpOffsetCv = BP_OFFSET_PARAM + att(BP_FREQ_ATT, BP_FREQ_INPUT)  // V/oct
bpTiltCv   =                    att(BP_TILT_ATT, BP_TILT_INPUT)  // V/oct (no knob)
distMode[n]= (int)roundf(BP{n}_DIST_MODE_PARAM)   // per band; 0=SOFT, 1=HARD, 2=FOLD
mix        = BP_MIX_PARAM

// Per-group CV (same for L and R base; tilt applied inside loop)
for n in {1, 2, 3}:
    freqCv[n] = bpOffsetCv + BP{n}_FREQ_PARAM + att(BP{n}_FREQ_ATT, BP{n}_FREQ_INPUT)
    focusCv[n] = BP{n}_FOCUS_PARAM + att(BP{n}_FOCUS_ATT, BP{n}_FOCUS_INPUT)
    driveCv[n] = BP{n}_DIST_PARAM  + att(BP{n}_DIST_ATT,  BP{n}_DIST_INPUT)

// BP section input: ALT path if patched, else LP1 output
bpInL = ALT_BP_L connected ? altL : lp1L
bpInR = ALT_BP_R connected ? altR : lp1R
```

**Per-sample processing:**
```
// Stereo tilt: L gets +tilt, R gets −tilt
for n in {1, 2, 3}:
    bpL[n] = SVFGroup[n].process(bpInL, freqCv[n] + bpTiltCv, focusCv[n], fs)
    bpR[n] = SVFGroup[n].process(bpInR, freqCv[n] − bpTiltCv, focusCv[n], fs)

    // Distortion (mode shared; drive per group)
    distL[n] = Distortion::processNorm(bpL[n] / 5, driveCv[n], distMode) × 5
    distR[n] = Distortion::processNorm(bpR[n] / 5, driveCv[n], distMode) × 5

// Sum all groups
wetL = clamp(distL[1] + distL[2] + distL[3], −10.5, +10.5)
wetR = clamp(distR[1] + distR[2] + distR[3], −10.5, +10.5)
```

**BP3 output tap (post-distortion, pre-mix):**
```
BP3_L_OUTPUT = distL[3]
BP3_R_OUTPUT = distR[3]
```

**BP_MIX (additive — dry at unity, wet added on top):**
```
// Dry is always LP1 output at unity. Wet is added. Hardware inverting summer behavior.
bpOutL = clamp(lp1L + wetL × mix, −12, +12)
bpOutR = clamp(lp1R + wetR × mix, −12, +12)
```

**Distortion modes** (input `x` and output `y` normalized to ±1; `d` = driveCv[n] ∈ [0, 1]):

- **SC (SOFT)** — diode chain; always clips to ±Vth = ±0.28 (±1.4 V):
```
G = driveParam ≤ 0.20 ? (driveParam/0.20) : exp((driveParam−0.20)/0.80 × 4)
y = Vth × tanh(G × x / Vth)    Vth = 0.28
```

- **HC (HARD)** — zener + diode clamp (±1.16 = ±5.8 V/5 V):
```
y = clamp(g × x, −1.16, +1.16)    // ±5.8V/5V (zener + diode Vf)
```

- **WF (FOLD)** — triangle wavefolder at ±1.4 V:
```
Vth = 0.28    // 1.4V/5V; two 1N4148W per polarity
y = (1 + d × 4) × x
output = Vth × asin(sin(π/2 / Vth × y)) × 2/π    // folds at ±1.4V, ~18 folds at max
```

### Block HP
```
hpFreqCv = HP_FREQ_PARAM + att(HP_FREQ_ATT, HP_FREQ_INPUT)
hpResCv  = HP_RES_PARAM  + att(HP_RES_ATT,  HP_RES_INPUT)
hpOutL   = HPFilter::process(bpOutL, hpFreqCv, hpResCv, sampleRate)
hpOutR   = HPFilter::process(bpOutR, hpFreqCv, hpResCv, sampleRate)
```

### Block LP2
```
lp2FreqCv = LP2_FREQ_PARAM + att(LP2_FREQ_ATT, LP2_FREQ_INPUT)
lp2ResCv  = LP2_RES_PARAM  + att(LP2_RES_ATT,  LP2_RES_INPUT)
lp2OutL   = LPFilter::process(hpOutL, lp2FreqCv, lp2ResCv, sampleRate)
lp2OutR   = LPFilter::process(hpOutR, lp2FreqCv, lp2ResCv, sampleRate)
```

### Block B — Output
```
MAIN_L_OUTPUT = clamp(lp2OutL, −11, +11)
MAIN_R_OUTPUT = clamp(lp2OutR, −11, +11)
LFO1_OUTPUT   = lfo1V
LFO2_OUTPUT   = lfo2V
```

## 7. CV / Attenuverter Routing

All CV destinations follow the same helper pattern (identical to old `modDest()`):

```cpp
// Returns: att_param × (cv_input connected ? cv_voltage : busV)
float attDest(float att, InputId cvIn, float busV);
```

The mod bus (`busV`) normalises into every CV input that is unpatched. Attenuverter trimpots
scale the result (−1 to +1, bipolar). This is unchanged from the old architecture.

### Changes to `Pogo.cpp` specifically:
- Rename/replace all old `ParamId`, `InputId`, `OutputId`, `LightId` enums (see §2–5)
- Update `configParam` / `configInput` / `configOutput` / `configLight` calls to match
- `process()` rewrite following §6 signal chain exactly
