POGO · WHAT HAPPENS ON COMMIT

← docs  ·  the CI pipeline behind every push

Push a change/<slug> branch and CI (.github/workflows/build.yml) runs on every push and PR:

  1. 1 · Five --check gates — components.py, fetch_datasheets.py, build_components.py (BOM + footprint SVGs + the docs viewer data), generate_schematic.py (pin coverage + structural + byte-drift), and build_panel.py (panel DRC). A stale generated artifact fails the build.
  2. 2 · Plugin builds — the .vcvplugin is cross-compiled for Linux, Windows and macOS. There is no Rack runtime in CI, so the plugin itself is verified by you in VCV Rack (download the artifact from the run).
  3. 3 · Artifacts — each run publishes the .vcvplugin per OS plus a POGO-hardware bundle: per-block schematics, the BOM (pogo-bom.csv), the panel SVG, and the panel debug HTML. Grab them from the workflow run's Artifacts.

Merge flow

  • PR change/<slug> → dev, squash-merged once CI is green.
  • dev → main promotes a release.
  • Never commit directly to dev or main.

For the full workflow — lanes, gates, and change records — see Change Process.
