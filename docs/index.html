What happens when you commit

Push a change/<slug> branch and CI (.github/workflows/build.yml) runs:

  • Five --check gates — components.py, fetch_datasheets.py, build_components.py (BOM + footprint SVGs + this page's data), generate_schematic.py (pin coverage + structural + byte-drift), build_panel.py (panel DRC). A stale generated artifact fails the build.
  • Plugin builds — cross-compiled .vcvplugin for Linux, Windows and macOS (there's no Rack runtime in CI, so you verify it in VCV Rack).
  • Artifacts — each run publishes the .vcvplugin per OS and a POGO-hardware bundle (schematics + BOM + panel SVG + debug HTML).

Then a PR change/<slug> → dev, squash-merged on green CI; dev → main promotes releases. See Change Process for the full workflow.
