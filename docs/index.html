SPACE COAST SYNTHESIZERS

Interactive Panel Editor →
Drag, rotate, add and delete panel components; toggle layers; live DRC; export a comment-preserving panel-data.yaml. Built from the source layout.
Plugin Topology →
Authoritative 48HP plugin spec — parameters, I/O, and DSP signal chain.
Plugin Builds (download) →
Linux / Windows / macOS .vcvplugin from CI. Open the latest green run → Artifacts → POGO-linux-x64 / -win-x64 / -mac. (GitHub login required; artifacts kept ~30 days.)
Plugin Source (DSP) →
The ground-truth software prototype: Pogo.cpp (module) and dsp/*.hpp (per-block DSP). Specs and schematics are reverse-engineered from it.
Bill of Materials →
Interactive BOM — 476 parts with MPN, footprint, and datasheet. View the whole project or group by block; search and sort.
Schematics →
Per-block KiCad schematics (10 blocks + shared-Q), generated from the netlists and rendered in-browser by GitHub. See SCHEMATIC-GEN-PLAN.md.
Netlist Visualizer →
Interactive force-directed view of the whole netlist — components as real KiCad footprints (passives as schematic glyphs) with pad-to-pad ratsnest. Toggle/collapse blocks, hide power as flags, marquee-select for inter-block boundary signals. A comprehension view, not authoritative.
Front Panel →
48HP faceplate artwork (generated from panel-data.yaml), embedded for viewing. Edit it live in the Panel Editor above.
PCB Layout Phase 5R · pending →
Board layout (placement + routing → Gerbers) is in progress; no .kicad_pcb files yet. Current architecture notes.
Change Process →
How the project is modified: plugin-first lanes (A/B/C), change/<slug> branches, changes/NNNN-slug.md records, and the Step 0–8 / G1–G6 gates.
What Happens on Commit →
The CI pipeline behind every push: the five --check gates, cross-platform plugin builds, and the artifacts each run publishes.
