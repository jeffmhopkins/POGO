# Block 6: Triple Bandpass + Distortion
Three independent 2-pole OTA-C SVF bandpass resonators with per-band frequency, Q (FOCUS), stereo spread (TILT), drive, and distortion mode. A global BP master section controls overall offset, tilt, bypass level, and wet level.

DSP source: `plugin/src/dsp/BandpassSVF.hpp`, `plugin/src/dsp/Distortion.hpp`, `plugin/src/Pogo.cpp` (lines 410вҖ“510)

---

## 1. Intent

Three bandpass resonators вҖ” each freely tunable over ~50 HzвҖ“3.2 kHz (F_REF = 400 Hz at param = 0). Together they act as a resonator bank, vowel sculptor, or harmonic comb filter. All three bands default to the same center frequency; the user positions them independently via the per-band FREQ knobs.

**Per-band signal order (change 0018, matching the locked plugin `Pogo.cpp:443вҖ“451`): DISTORTION *before* the bandpass.** Each band is `input вҶ’ DISTn (drive + SC/HC/WF) вҶ’ SVFn (2-pole bandpass) вҶ’ band output`; the three band outputs are then summed and dry/wet-mixed. (BP1/BP2 input = LP1 band; BP3 input = the ALT-VCA-or-LP1 selector вҖ” see ALT path.) The hardware nets carry this as `LP1_OUT/BP3IN вҶ’ distN вҶ’ BPn_DISTOUT вҶ’ svfN вҶ’ BPn_BANDOUT/BP3_TAP вҶ’ mix`.

### Global BP master controls

**BP_OFFSET** shifts all three bands simultaneously (master V/oct offset). **BP_TILT** creates global stereo spread вҖ” L channel gets +tiltV, R gets вҲ’tiltV вҖ” separating the band positions across the stereo field.

**BP_BYPASS** sets the level of the pre-BP signal (LP1 output or ALT input) that passes through to the BP output (0 = muted, 1 = full). **BP_WET** sets the level of the processed (post-distortion) BP sum at the output (0 = dry only, 1 = full wet). The output formula is:

```
bpOut = clamp(bandIn Г— bypass + wetSum Г— wet, вҲ’12, +12)
```

Both controls are independent scalers вҖ” not a crossfade. Setting both to 1.0 adds LP1 and full BP together; setting BYPASS to 0 gives pure BP; setting WET to 0 gives pure LP1 passthrough.

CV inputs for the master section: FREQ jack (+ attenuverter trimpot) modulates BP_OFFSET; TILT jack modulates BP_TILT.

### Per-band controls (BP1, BP2, BP3 вҖ” identical architecture)

Each band has:
- **DIST switch** (Dailywell DW5, 3-position toggle): Soft Clip (down) / Hard Clip (centre) / Wavefold (up) вҖ” selects distortion character independently per band
- **FREQ knob** (xl): per-band frequency offset in V/oct; F_REF = 400 Hz вҶ’ range ~50 HzвҖ“3.2 kHz
- **FOCUS knob** (large): resonance (Q); `Q = 0.5 Г— 400^focus` вҶ’ Q range 0.5вҖ“200; does not self-oscillate
- **TILT knob** (large): per-band stereo spread, additive with global BP_TILT
- **DIST knob** (large): per-band drive depth (0вҖ“1) into the distortion cell, which sits **before** the bandpass (0.20 = unity/threshold)
- **CLIP LED**: illuminates when the per-band distortion output (pre-bandpass) exceeds Вұ4 V
- CV inputs per band: FREQ, TILT, DIST jacks (each with attenuverter trimpot)

### ALT path

`ALT_BP_L/R` jacks (with the `ALT_GAIN` 1Г—/5Г— switch) feed **BP3 only**, and the ALT signal still
passes **through the VCA** (same control as the main path) вҖ” it bypasses **LP1**, not the VCA (see
block-1/block-4). A per-channel selector (block-6, BP3) routes the ALT-VCA signal (block-4
`ALT_VCA_OUT_L/R`) into BP3's input when the ALT jacks are patched, else BP3 takes the LP1 band.
BP1 and BP2 always take the LP1 band.

### BP3 output tap

`BP3_L_OUT` and `BP3_R_OUT` jacks provide BP3's **post-bandpass band output** (the filtered BP3
voice, after its distortion+SVF, before the 3-band sum and dry/wet mix) вҖ” useful for parallel
processing or sidechain routing. R normals to L when the R jack is unpatched.

### SVF Groups (3 independent resonators)

Each group is a 2-pole state-variable filter (BP output only). Both DSP and hardware use one 2-pole OTA-C SVF per group per channel:

```
H_BP(s) = (ПүвӮҖ/Q В· s) / (sВІ + ПүвӮҖ/Q В· s + ПүвӮҖВІ)
```

Peak at s = jПүвӮҖ: |H_BP(jПүвӮҖ)| = 1 (constant unity, regardless of Q). Q controls bandwidth only, not peak amplitude.

**f_ref per group** (at 0V CV): **all three bands = 400 Hz** (matches the locked plugin
`BandpassSVF.hpp:42` `F_REF = {400, 400, 400}`). The bands default to the same center and the
player spreads them via the per-band FREQ knobs вҖ” they are not fixed-formant. (The earlier
`{200, 1500, 6000}` per-group formant assignment matched a stale code comment, not the plugin
code, and is superseded вҖ” change 0018. Integrator caps are now a uniform 68 nF C0G, trimmed to
400 Hz by each band's f_ref trim.)
| Group | f_ref | Cap |
|---|---|---|
| BP1 | 400 Hz | 68 nF (C15) |
| BP2 | 400 Hz | 68 nF (C19) |
| BP3 | 400 Hz | 68 nF (C23) |

**Frequency control:**
```
fвӮҖ = f_ref[i] Г— 2^(BP_OFFSET + BP_FREQ_i + BP_TILT Г— Вұ1)
```
where BP_TILT applies +tiltV on L channel, вҲ’tiltV on R channel.

**Q control:**
```
DSP: Q = 0.5 Г— 400^qParam  (range 0.5вҖ“200; does NOT self-oscillate by design)
Hardware: Q = 52mV / (Iabc Г— R_in)
```
BP Q maximum is ~200 at hardware (Iabc вҶ’ minimum before instability threshold). Self-oscillation is suppressed вҖ” BP groups are resonators, not oscillators.

### Distortion

Three modes selected by `BP_DIST` (global switch, 0/1/2):

**Soft Clip (mode 0):**
```
G = driveParam вүӨ 0.20 ? (driveParam / 0.20)        (0вҶ’1Г— gain control)
      : exp((driveParam вҲ’ 0.20) / 0.80 Г— 4)         (1Г—вҶ’~55Г— drive, same exp law)
y = Vth Г— tanh(G Г— x / Vth)    Vth = 0.28 (= 1.4V/5V; two 1N4148W per polarity)
```
Output bounded to ВұVth = Вұ1.4V at all drive levels. Linear below threshold; smooth tanh onset above. Diodes always in circuit; gain-control zone (вүӨ9am) feeds the diode chain at reduced gain.

**Hard Clip (mode 1):**
```
g = 1 + d Г— 4    (1вҖ“5Г— linear gain)
y = clamp(g Г— x, вҲ’1.16, +1.16)    (Вұ1.16 = Вұ5.8V/5V; zener 5.1V + 1N4148W Vf 0.7V)
```
Linear gain into hard rails at Вұ5.8V. Aggressive, brick-wall limiting.

**Wavefold (mode 2, Buchla-style):**
```
Vth = 0.28    (= 1.4V/5V; two 1N4148W per polarity in passive clamp)
y = (1 + d Г— 4) Г— x                    (1вҖ“5Г— pre-gain)
output = Vth Г— asin(sin(ПҖ/2 / Vth Г— y)) Г— 2/ПҖ
```
Passive diode clamp at ВұVth, then V_out = 2Г—V_clamp вҲ’ V_in (slope reversal at Вұ1.4V). At max drive and full-scale input: ~18 folds vs ~3 folds at Вұ5V threshold. Dense Buchla-style folding.

### BP_MIX blend

```
BP_out = clamp(LP1_output + mix Г— sum(distorted_groups), вҲ’12, +12)
```
Dry is always present at unity. Wet adds on top. Mix=0: LP1 only. Mix=max: LP1 + full wet (up to 6 dB louder than a crossfade). Hardware op-amp rails limit output; DSP clamps at Вұ12V. Default mix=0.5.

### Oversampling

None. Hardware SVF and distortion run at audio rate; analog hardware is inherently alias-free. DSP matches: no oversampling applied to BP section.

### DSP/Hardware Alignment

DSP and hardware are aligned on all BP signal-processing behaviors:
- 2-pole SVF per group per channel (both)
- Unity BP peak gain regardless of Q (both)
- Additive BP_MIX: dry at unity, wet added on top (both)
- No oversampling (both)
- Distortion thresholds: SC Вұ1.4V, HC Вұ5.8V, WF Вұ1.4V (both)

вҶ’ See aux-ota-c-svf.md for SVF topology; aux-distortion.md for distortion cells.

### OTA-C SVF instances

Three groups Г— two channels (L + R) = **6 OTA-C SVF instances**. Each instance requires:
- 1Г— LM13700 OTA cell as integrator (one IC = 2 cells = one group, one channel)
- 1Г— TL072 half as SUM_AMP

Grouping per IC:
- Per group: `BP_OTA_G1_L`, `BP_OTA_G1_R` etc. вҖ” one LM13700 per group per channel (dual OTA: cell A + cell B = integrator + Q VCA, OR cell A/B for two channels of same group)
- Conservative count: 3 groups Г— 2 channels Г— 1 LM13700 = **6Г— LM13700** for integrators + Q VCAs

**Q (resonance) at SUM_AMP:**
Standard SVF BP peak is 1Г— (constant unity, independent of Q). No normalization is applied. Q controls bandwidth only; high-Q formants have the same peak level as low-Q formants. This is the natural hardware OTA-C SVF behavior and is preserved in the DSP.

**Stereo tilt implementation:**
- BP_TILT generates +V_tilt (from mod bus via attenuverter)
- +V_tilt added to L-channel expo converter offset sum
- вҲ’V_tilt generated by TL072 inverting unity buffer; added to R-channel expo converter offset sum
- One TL072 half used as tilt inverter (shared across all 3 groups, same tilt offset applied to all)

**Expo converters:**
- **Per-channel** THAT340: one per L and one per R, per group вҖ” 6Г— THAT340 total (U28-U30 L, U70-U72 R)
- Each band/channel has its own f_ref trim (RV_BP{1,2,3}_REF L/R), all targeting **400 Hz at 0 V**

**Distortion hardware (see aux-distortion.md):**
- Three sub-circuit chains per group: SC / HC / WF all running simultaneously
- CD4053 triple 2-channel analog mux per group selects which mode's output passes
- All 3 CD4053 select pins tied together вҶ’ BP_DIST switch controls all groups simultaneously
- One Dailywell DW5 (2M DPDT ON-ON-ON) toggle вҶ’ its two poles give the 2 select lines to all 3 CD4053 ICs
- Distortion runs post-SVF at audio rate (no oversampling in analog hardware)
- BP3 output tap: taken at group 3's **post-bandpass** band output (`BP3_TAP`, after DIST3вҶ’SVF3), available at BP3_L/R_OUT jacks

**Oversampling:**
- None. Hardware and DSP both run at base sample rate. Analog hardware is inherently alias-free; no oversampling required.

**Board assignment:** BP SVF and distortion on audio board. BP control pots and jacks on control board. THAT340s on audio board near their respective SVF clusters.

### Q Normalization вҖ” Design Decision

Hardware OTA-C SVF produces |H_BP(jПүвӮҖ)| = 1 (constant unity peak). Q controls bandwidth only. A 1/QВІ normalization would require a VCA per group per channel (6 cells), significantly increasing complexity. Decision: hardware unity peak is preserved in both hardware and DSP. High-Q formants are louder вҖ” this is musically desirable (tight resonances stand out) and is the expected behavior of analog formant filters.

### Distortion Sub-Circuit Design (SC / HC / WF)

One set of three sub-circuit paths (SC + HC + WF) per group, per channel = 6 sets total.
All paths share a common DRIVE control signal per group (one pot + CV per group, summed at
mod bus). All three paths run continuously; the CD4053 selects which output reaches BP_MIX.

**Transfer function:**
```

```

This is a true slope reversal at Вұ1.4 V, matching the asin(sin(x)) shape for a single fold.
At DRIVE=max (gain 5.7Г—), a Вұ5V input signal reaches Вұ28V pre-fold вҶ’ multiple reflections
across the Вұ1.4V window вҶ’ dense odd-harmonic spectrum.

**Stability:** TL072-B is in standard negative feedback (output вҶ’ R_f вҶ’ (вҲ’) input).
V_clamp at (+) is a passive resistor-diode network with no active elements; it presents a
source impedance of R_clamp = 10 kО© at (+), which has no impact on loop stability.
Phase margin is identical to a standard G=+2 non-inverting configuration. No stability
concern вҖ” no prototype verification required for this stage.

**Diode Vth variation (resolved вҖ” accepted characteristic):**

1N4148W Vf depends on clamp current; Vth = 2Г—Vf shifts with drive level:

```
V_fold_in   I_clamp = (VвҲ’1.4V)/10kО©   Vf (1N4148W)   Vth = 2Г—Vf
  3 V           160 ВөA                  ~0.62 V          ~1.24 V
  5 V           360 ВөA                  ~0.64 V          ~1.28 V
 10 V           860 ВөA                  ~0.68 V          ~1.36 V
 28 V (max)    2.66 mA                  ~0.72 V          ~1.44 V
```

At practical folding depths (V_fold_in > 5 V), Vth sits in the 1.28вҖ“1.44 V range,
within Вұ10% of the DSP target (1.4 V). At very low drive (barely past threshold),
Vth is lower (~1.2 V), which softens fold onset slightly. This gives the hardware a
characteristically smooth entry into folding вҖ” the fold corners are rounder than the
mathematical `asin(sin(x))` model, consistent with all passive-diode Buchla-style
folders. It is an analog character feature, not a defect. No hardware change required.

Components per WF path: 2Г— TL072 halves (pre-gain + folder; same IC, no additional ICs),
4Г— 1N4148W (SOD-123), 4Г— resistors (R_WF_in, R_WF_fb_fixed, R_clamp, R_g = R_f = 10 kО©).

**Note on wiper bypass caps:** A 47 pF cap from each RV_DRIVE wiper to AGND is recommended
(pole at R_wiper_max Г— C вүҲ 11.75 kО© Г— 47 pF вҶ’ f_pole вүҲ 288 kHz). Kills RF pickup; no
effect on control-rate drive changes.

### CD4053 Mux Wiring

One CD4053 (SOIC-16) per group, 3 total (BP_DIST_MUX_1, _2, _3). All three have their
S_A and S_B control pins tied together (global mode select from SW_DIST panel switch).

**Power supply wiring:**
```
VDD (+12V supply) вҶ’ pin 16 of each CD4053 (analog/digital supply)
VSS (вҲ’12V supply) вҶ’ pin 7 of each CD4053 (negative analog rail)
VEE (GND)         вҶ’ pin 8 of each CD4053 (digital ground; sets logic threshold at ~VDD/2)
INH pin (pin 6)   вҶ’ GND (active-low inhibit; tied LOW = always enabled)
```

The CD4053 passes signals from VSS to VDD on each analog channel. With Вұ12V supply,
the audio signal can swing Вұ5V without signal-dependent R_on distortion.

**Control line encoding (2 binary lines вҶ’ 3 modes):**
```
SW_DIST position вҶ’ S_A line, S_B line:
  Soft Clip:  S_A = 0,  S_B = 0   вҶ’ Channel X passes SC_out; Channel Y passes X-output
  Hard Clip:  S_A = 1,  S_B = 0   вҶ’ Channel X passes HC_out; Channel Y passes X-output
  Wavefold:   S_A = x,  S_B = 1   вҶ’ Channel Y selects WF_out (overrides Channel X)
```

**Channel assignment within each CD4053:**
```
Channel X (control S_A):
  X0 input вҶҗ SC_out
  X1 input вҶҗ HC_out
  X output вҶ’ Channel Y input Y0

Channel Y (control S_B):
  Y0 input вҶҗ Channel X output
  Y1 input вҶҗ WF_out
  Y output вҶ’ DIST_out (вҶ’ BPn_DISTOUT вҶ’ SVF input; the SVF band output then goes to the mix)

Channel Z: spare (inputs tied to GND; output unused)
```

When S_B = 1: Channel Y selects WF regardless of S_A вҖ” wavefold mode overrides.
When S_B = 0: Channel Y passes Channel X output, which is SC or HC depending on S_A.

SW_DIST is a Dailywell DW5 (2M DPDT ON-ON-ON) toggle generating S_A and S_B:
- Position 0 (Soft):  S_A=low, S_B=low   (0V = logic LOW; from GND)
- Position 1 (Hard):  S_A=high, S_B=low  (+5V = logic HIGH; from +5V regulator or +12V with R+zener)
- Position 2 (Fold):  S_A=low, S_B=high

The +5V logic rail for CD4053 control inputs: taken from a 78L05 regulator (or 5.1V zener
from +12V with 1kО© series R) on the audio board. Logic signal swings 0VвҖ“5V, which is within
the CD4053 VIH threshold at VDD=+12V.

### BP_MIX Blend Circuit

The DSP: `V_out = (1 вҲ’ mix) Г— V_dry + mix Г— V_wet`  
where V_dry = LP1 signal (entering block-6), V_wet = sum of three distorted BP groups.

**BP summing stage (V_wet):**

One TL072 inverting summer combines distortion outputs of all three groups (per channel):
```
DIST_BP1_out в”Җв”Җ[R_sum = 33 kО©]в”Җв”Җв”җ
DIST_BP2_out в”Җв”Җ[R_sum = 33 kО©]в”Җв”Җв”јв”Җв”Җв–ә(вҲ’) TL072 в”Җв”Җ[R_f = 33 kО©]в”Җв”Җв–ә V_wet_inv
DIST_BP3_out в”Җв”Җ[R_sum = 33 kО©]в”Җв”Җв”ҳ         в”Ӯ
                                         (+) = GND
```
V_wet_inv = вҲ’(BP1 + BP2 + BP3) / 3 (normalized). Signal inverted; corrected at the fixed
output polarity-restore stage (U27 half B).

**BP_MIX summing amp:**

A TL072 inverting summer with the MIX pot controlling the wet-to-dry ratio:
```
V_dry в”Җв”Җ[R_dry = 100 kО©]в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
                                                  в”ңв”Җв”Җв–ә(вҲ’) TL072 в”Җв”Җ[R_mix_f = 100 kО©]в”Җв”Җв–ә V_mix_inv
V_wet_inv в”Җв”Җ[R_wet_pot = RV_BP_MIX wiper Г— 100k]в”Җв”ҳ
                                                (+) = GND
```

RV_BP_MIX (linear-taper 100 kО©): controls R_wet from 0 (full CCW, no wet) to 100 kО© (full CW).
V_mix_inv = вҲ’(V_dry Г— R_f/R_dry + V_wet_inv Г— R_f/R_wet)

At MIX=0 (R_wet = вҲһ): V_mix_inv = вҲ’V_dry (dry inverted)
At MIX=max (R_wet = 100k, R_f = R_dry = 100k):
  V_mix_inv = вҲ’V_dry вҲ’ V_wet_inv = вҲ’V_dry вҲ’ (вҲ’V_wet) = вҲ’V_dry + V_wet

**Wet polarity restore (U48):** An inverting unity-gain buffer on the V_wet_inv signal
restores wet polarity to +V_wet before the MIX amp, so V_wet_pos = вҲ’V_wet_inv = +V_wet.

With V_wet_pos (corrected) feeding the MIX amp instead of V_wet_inv:
  V_mix_inv = вҲ’(V_dry + V_wet_pos Г— R_f/R_wet)
At MIX=0: V_mix_inv = вҲ’V_dry
At MIX=max: V_mix_inv = вҲ’V_dry вҲ’ V_wet

V_mix_inv then passes to the fixed output polarity-restore stage (U27 half B).

**DESIGN NOTE вҖ” U48 reassigned:** U48 (previously "output polarity buffer") is used as
the wet polarity restorer. This inserts one inversion in the wet path to correct the
polarity before the MIX amp. Without this correction, the wet signal would be subtracted
rather than added at the BP output (V_dry вҲ’ V_wet instead of V_dry + V_wet).

**Note:** DSP matches hardware: dry is always present at unity, wet is added on top. At MIX=max: output = dry + wet (up to 6 dB louder than a crossfade at equal level). DSP clamps at Вұ12V; hardware limited by op-amp rails.

### BP output polarity restore (fixed)

The hardware MIX amp output V_mix_inv is inherently negative-polarity (вҲ’V_dry вҲ’ V_wet).
A fixed G=вҲ’1 inverter (U27 half B, in the BP_TILT_INV TL072CDT) restores the expected
positive-polarity BP output (+V_dry + V_wet). There is no user polarity control (BP_POL
was removed вҖ” the plugin has no polarity param; the inversion below is always in path).

```
                               R_pol_in (100 kО©)    R_pol_fb (100 kО©)
V_mix_inv в”Җв”Җ[R_pol_in = 100 kО©]в”Җв”Җв–ә(вҲ’) U27-B в”Җв”Җ[R_pol_fb]в”Җв”Җв–ә V_bp_out (+V_dry+V_wet)
                                        в”Ӯ
                                   (+) = GND
```

U27 (BP_TILT_INV TL072CDT) halves:
- Half A: generates вҲ’V_tilt for the R-channel expo converter (unchanged).
- Half B: fixed G=вҲ’1 inverter; restores V_bp_out polarity (always in path; no switch).

**Board assignment:** BP_MIX summing amp + output polarity-restore inverter on the audio board.
